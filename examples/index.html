Python

Code Reviewer Async: An example of using the async API to run a chain of messages asynchronously. It uses a git diff as the material for a code review. The results are a code review from a software engineer, a pull request description, and a description that could be used as a git commit message.

Comparing Code Models: This example uses a prompt with the GPT-3.5 and GPT-4 models that will generate suggestions for code improvements and then apply the suggestions to the code.

Describe Topic Experiment: Uses an experiment to compare the responses from GPT-3.5 and GPT-4 for a chain of prompts that asks

Hello World: Chain of two prompts; the first asks for the capital of a country, the second asks for the city's population.

Hello World Async: Same as Hello World, however it uses the async API to run two chains asynchronously. It asks for the capital cities of two countries and then asks for the population of each.

Hello World Experiment: Same as Hello World, however it uses an experiment to run the chain with two different models and different model parameters. Also prints out the output in different formats (CSV, JSON, Pandas DataFrame).

Interesting Facts: Demonstrates using methods to transform and modify prompts and to extract data from responses. It will prompt for a list of facts about a topic, which are formatted in a particular way, and then it extracts the first fact and passes it along to the next prompt. The response styles are changed between the prompts, the response length is added to the prompt, and the list format to be used is also part of the prompt.

Math: An example based on the math example from MiniChain. It uses few-shot training prompt to include math questions and the code that solves them. The prompt asks for the Python code to solve the given math problem and then runs the code to produce the answer.
